Maze Pathfinder Lab
OpenFrameworks 기반 미로 탐색 알고리즘 비교 시각화 프로젝트

============================================================
1. 프로젝트 개요
============================================================
본 프로젝트는 OpenFrameworks를 이용하여 BFS, DFS, Dijkstra, A* 알고리즘의
탐색 과정을 시각적으로 비교하는 인터랙티브 미로 탐색 시뮬레이터입니다.

단순히 최종 경로만 보여주는 것이 아니라, 각 알고리즘이 어떤 순서로 노드를
방문하는지 애니메이션으로 보여주며, 방문 노드 수, 경로 길이, 총 이동 비용,
실행 시간(ms)을 함께 출력합니다.

가중치 셀을 지원하기 때문에 BFS와 Dijkstra/A*의 차이를 명확하게 확인할 수
있습니다. 또한 마우스로 벽, 시작점, 도착점, 가중치 셀을 직접 편집할 수 있어
여러 case를 반복적으로 실험할 수 있습니다.

============================================================
2. 실행 환경
============================================================
- macOS + Xcode 또는 Windows + Visual Studio
- OpenFrameworks 설치 필요
- 추가 addon 없음
- C++ / OpenFrameworks 기본 라이브러리만 사용

권장 환경:
- openFrameworks 0.12.x 이상
- macOS 사용 시 Xcode + openFrameworks Project Generator 사용 권장

============================================================
3. 설치 및 실행 방법
============================================================
[macOS / Xcode 기준]

1) 압축을 풉니다.

2) MazePathfinderLab 폴더를 OpenFrameworks의 myApps 폴더 안에 넣습니다.
   예시:

   openFrameworks/apps/myApps/MazePathfinderLab

3) openFrameworks Project Generator를 실행합니다.

4) Import 버튼으로 아래 폴더를 선택합니다.

   openFrameworks/apps/myApps/MazePathfinderLab

5) Addons는 추가하지 않습니다.

6) Update 버튼을 눌러 Xcode 프로젝트 파일을 생성합니다.

7) 생성된 MazePathfinderLab.xcodeproj를 Xcode로 열고 실행합니다.


[터미널 make 실행이 가능한 경우]

1) 다음 경로로 이동합니다.

   cd openFrameworks/apps/myApps/MazePathfinderLab

2) 빌드 및 실행합니다.

   make
   make RunRelease

주의:
- OpenFrameworks는 설치 경로와 버전에 따라 Xcode 프로젝트 파일을 새로 생성하는 방식이 가장 안전합니다.
- 따라서 본 압축 파일에는 소스코드, 데이터 파일, Makefile, config.make, addons.make를 포함했습니다.
- Xcode 프로젝트 파일은 Project Generator로 생성하면 됩니다.

5. 조작 방법
============================================================
1 : BFS 실행
2 : DFS 실행
3 : Dijkstra 실행
4 : A* 실행

M : 마우스 편집 모드 변경
    Wall Toggle  -> 벽 추가/삭제
    Move Start   -> 시작점 이동
    Move Goal    -> 도착점 이동
    Weight Toggle-> 가중치 셀 추가/삭제

L : 다음 샘플 미로 파일 로드
G : 랜덤 미로 생성
C : 빈 격자 미로 생성
R : 탐색 결과 초기화
P : 애니메이션 일시정지/재개
+ : 애니메이션 속도 증가
- : 애니메이션 속도 감소
H : 도움말 표시/숨김

마우스 클릭/드래그 : 현재 편집 모드에 따라 셀 수정

7. 구현 알고리즘
============================================================
1) BFS
- 자료구조: queue<Node>
- 목적: 가중치가 없는 미로에서 시작점으로부터 가까운 칸부터 탐색
- 특징: 모든 이동 비용이 같을 때 최단 칸 수 경로 보장
- 시간복잡도: O(V + E)
- 공간복잡도: O(V)

2) DFS
- 자료구조: stack<Node>
- 목적: 한 방향으로 최대한 깊이 탐색한 뒤 되돌아가는 방식 시각화
- 특징: 경로를 찾을 수는 있지만 최단 경로를 보장하지 않음
- 시간복잡도: O(V + E)
- 공간복잡도: O(V)

3) Dijkstra
- 자료구조: priority_queue<PriorityState>
- 목적: 가중치가 있는 미로에서 총 이동 비용이 가장 작은 경로 탐색
- 특징: 가중치 셀을 고려한 최적 비용 경로 보장
- 시간복잡도: O((V + E) log V)
- 공간복잡도: O(V)

4) A*
- 자료구조: priority_queue<PriorityState>
- 목적: g(n) + h(n)을 이용하여 도착점 방향으로 효율적으로 탐색
- 휴리스틱: Manhattan Distance
- 특징: Dijkstra보다 적은 노드를 방문하는 경우가 많으며, 본 격자 환경에서 최적 경로 탐색 가능
- 시간복잡도: 최악 O((V + E) log V)
- 공간복잡도: O(V)

여기서 V는 미로의 칸 수(rows * cols), E는 인접 이동 가능한 간선 수입니다.
격자 미로에서는 각 칸이 최대 4개의 이웃을 가지므로 E는 대략 4V 이하입니다.

8. 주요 자료구조
============================================================
struct Node
- row, col 좌표 저장
- 알고리즘에서 현재 위치, 이웃 위치, 부모 위치를 표현

struct Cell
- CellType type: 빈 칸, 벽, 시작점, 도착점, 가중치 셀 구분
- int cost: 이동 비용 저장

vector<vector<Cell>> grid
- 전체 미로를 2차원 배열 형태로 저장하는 핵심 자료구조

vector<vector<bool>> visibleVisited
- 애니메이션에서 현재까지 화면에 표시된 방문 노드 여부 저장

vector<vector<bool>> visiblePath
- 애니메이션에서 현재까지 화면에 표시된 최종 경로 여부 저장

vector<vector<Node>> parent
- 각 노드가 어디에서 왔는지 저장하여 최종 경로를 역추적하는 데 사용

queue<Node>
- BFS에서 사용

stack<Node>
- DFS와 랜덤 미로 생성에서 사용

priority_queue<PriorityState>
- Dijkstra와 A*에서 최소 비용 또는 최소 f-cost 노드 선택에 사용

============================================================
9. 주요 함수
============================================================
loadMazeFromFile()
- txt 미로 파일을 읽어 grid에 저장합니다.

runBFS()
- queue 기반 BFS를 수행하고 방문 순서와 최종 경로를 반환합니다.

runDFS()
- stack 기반 DFS를 수행하고 방문 순서와 최종 경로를 반환합니다.

runDijkstra()
- priority_queue 기반 Dijkstra 알고리즘을 수행합니다.

runAStar()
- Manhattan Distance 휴리스틱을 사용하는 A* 알고리즘을 수행합니다.

reconstructPath()
- parent 배열을 따라가며 도착점에서 시작점까지의 경로를 복원합니다.

generateRandomMaze()
- stack을 이용한 Recursive Backtracking 방식으로 랜덤 미로를 생성합니다.

drawMaze(), drawSidePanel(), drawLegend()
- 미로와 통계 패널, 범례를 화면에 출력합니다.

applyEditToCell()
- 현재 EditMode에 따라 마우스로 선택한 셀을 벽, 시작점, 도착점, 가중치로 변경합니다.

============================================================
10. 발표/보고서에 강조할 창의적 구현 항목
============================================================
- BFS, DFS, Dijkstra, A* 네 가지 알고리즘을 한 프로그램에서 비교 가능
- 단순 미로 탐색이 아니라 방문 순서 애니메이션 제공
- 가중치 셀을 통해 BFS와 비용 기반 알고리즘의 차이 시각화
- 방문 노드 수, 경로 길이, 총 비용, 실행 시간 통계 제공
- 마우스 기반 벽/시작점/도착점/가중치 편집 지원
- 랜덤 미로 생성 기능 지원
- queue, stack, priority_queue, 2D vector 등 자료구조 활용
