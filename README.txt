Maze Pathfinder Lab
OpenFrameworks 기반 미로 탐색 알고리즘 비교 시각화 프로젝트

============================================================
1. 프로젝트 개요
============================================================
본 프로젝트는 OpenFrameworks를 이용하여 BFS, DFS, Dijkstra, A* 알고리즘의
탐색 과정을 한 화면에서 비교할 수 있도록 만든 인터랙티브 미로 탐색 시뮬레이터입니다.

단순히 최종 경로만 출력하는 프로그램이 아니라, 각 알고리즘이 어떤 순서로 칸을
방문하는지 애니메이션으로 보여주고, 방문 노드 수, 최종 경로 길이, 총 이동 비용,
실행 시간(ms)을 함께 출력합니다. 따라서 알고리즘의 이론적 차이를 시각적으로
확인할 수 있습니다.

가중치 셀을 지원하기 때문에 BFS처럼 간선 비용을 고려하지 않는 탐색과,
Dijkstra/A*처럼 비용을 고려하는 탐색의 차이를 명확하게 확인할 수 있습니다.
또한 마우스로 벽, 시작점, 도착점, 가중치 셀을 직접 편집할 수 있어 사용자가
여러 실험 case를 직접 만들고 반복적으로 비교할 수 있습니다.

본 프로젝트의 핵심 목표는 다음과 같습니다.
- 그래프 탐색 알고리즘을 실제 미로 문제에 적용한다.
- queue, stack, priority_queue, 2D vector, parent 배열 등 자료구조를 명확히 활용한다.
- 최종 경로뿐 아니라 방문 순서를 시각화하여 알고리즘의 동작 원리를 설명 가능하게 한다.
- 가중치 셀을 통해 최단 거리와 최소 비용의 차이를 실험적으로 보여준다.
- 결과 통계를 제공하여 시간복잡도와 공간복잡도 설명을 보고서/발표에 연결한다.

2. 실행 환경
============================================================
필수 환경:
- C++
- OpenFrameworks 설치
- macOS + Xcode 또는 Windows + Visual Studio
- 추가 addon 없음
- OpenFrameworks 기본 라이브러리만 사용

권장 환경:
- openFrameworks 0.12.x 이상
- macOS 사용 시 Xcode + OpenFrameworks Project Generator 사용 권장

확인된 실행 방식:
- macOS 터미널에서 make 빌드 가능
- make 후 make RunRelease 또는 생성된 실행 파일 직접 실행 가능

3. 설치 및 실행 방법
============================================================
[macOS / Xcode 기준]

1) 압축 파일을 풉니다.

2) MazePathfinderLab 폴더를 OpenFrameworks의 myApps 폴더 안에 넣습니다.
   예시:

   openFrameworks/apps/myApps/MazePathfinderLab

3) OpenFrameworks Project Generator를 실행합니다.

4) Import 버튼으로 아래 폴더를 선택합니다.

   openFrameworks/apps/myApps/MazePathfinderLab

5) Addons는 추가하지 않습니다.

6) Update 버튼을 눌러 Xcode 프로젝트 파일을 생성합니다.

7) 생성된 MazePathfinderLab.xcodeproj를 Xcode로 열고 실행합니다.


[터미널 make 실행이 가능한 경우]

1) 프로젝트 폴더로 이동합니다.

   cd openFrameworks/apps/myApps/MazePathfinderLab

2) 빌드합니다.

   make

3) 실행합니다.

   make RunRelease

또는 빌드 완료 메시지에 표시되는 실행 파일 경로로 직접 이동하여 실행할 수 있습니다.

주의:
- 명령어는 대소문자를 구분합니다. make RunRelease가 맞고, make Runrelease는 잘못된 명령입니다.
- OpenFrameworks는 설치 경로와 버전에 따라 Xcode/Visual Studio 프로젝트 파일을 새로 생성하는 방식이 가장 안전합니다.
- 따라서 제출 압축 파일에는 src 폴더, data 파일, Makefile, config.make, addons.make를 포함하고,
  Xcode/Visual Studio 프로젝트 파일은 Project Generator로 재생성하는 방식을 권장합니다.

6. 미로 파일 형식
============================================================
bin/data 폴더 안의 txt 파일을 읽습니다.

사용 가능한 문자:
# : 벽
. : 빈 칸
S : 시작점
G : 도착점

W : 이동 비용 5인 가중치 셀

예시:
#########
#S..2...#
#.###.#G#
#...W...#
#########

설명:
- S는 시작점입니다.
- G는 도착점입니다.
- #은 탐색할 수 없는 벽입니다.
- .은 이동 비용 1의 일반 칸입니다.
- 2는 이동 비용 2의 가중치 칸입니다.
- W는 이동 비용 5의 가중치 칸입니다.

7. 구현 알고리즘 설명
============================================================
1) BFS
BFS는 queue를 사용하는 너비 우선 탐색입니다. 시작점에서 가까운 깊이의 칸부터
차례로 탐색하므로, 모든 칸의 이동 비용이 동일한 일반 미로에서는 최단 칸 수
경로를 보장합니다. 그러나 가중치 칸이 있는 경우 이동 비용을 고려하지 않으므로
총 비용이 최소인 경로를 항상 보장하지는 않습니다.

2) DFS
DFS는 stack을 사용하는 깊이 우선 탐색입니다. 한 방향으로 가능한 깊게 들어간 뒤
막히면 되돌아오는 방식으로 동작합니다. 경로를 빠르게 찾을 수는 있지만, 최단 경로나
최소 비용 경로를 보장하지 않습니다. 본 프로젝트에서는 BFS와 DFS의 방문 순서 차이를
애니메이션으로 비교할 수 있습니다.

3) Dijkstra
Dijkstra 알고리즘은 priority_queue를 사용하여 현재까지의 누적 이동 비용이 가장 작은
칸을 먼저 확장합니다. 모든 이동 비용이 0 이상이면 시작점에서 도착점까지의 최소 비용
경로를 구할 수 있습니다. 가중치 셀이 있는 미로에서 BFS와의 차이가 가장 잘 드러납니다.

4) A*
A* 알고리즘은 Dijkstra의 누적 비용 g(n)에 목표까지의 추정 거리 h(n)을 더한
f(n)=g(n)+h(n)을 기준으로 탐색합니다. 본 프로젝트에서는 상하좌우 이동 격자에 적합한
Manhattan Distance를 휴리스틱으로 사용합니다. 목표 방향 정보를 활용하기 때문에
많은 경우 Dijkstra보다 더 적은 노드를 방문하면서도 최소 비용 경로를 찾을 수 있습니다.

============================================================
8. 시간복잡도 및 공간복잡도
============================================================
V = 미로의 칸 수 = rows * cols
E = 인접 이동 가능한 간선 수
격자 미로에서는 각 칸이 최대 네 방향의 이웃을 가지므로 E는 대략 4V 이하입니다.

알고리즘별 복잡도:
- BFS
  시간복잡도: O(V + E)
  공간복잡도: O(V)
  이유: 각 칸과 간선을 최대 한 번씩 확인하며, queue와 visited 배열이 필요합니다.

- DFS
  시간복잡도: O(V + E)
  공간복잡도: O(V)
  이유: 각 칸과 간선을 최대 한 번씩 확인하며, stack과 visited 배열이 필요합니다.

- Dijkstra
  시간복잡도: O((V + E) log V)
  공간복잡도: O(V)
  이유: priority_queue에서 삽입/삭제가 log V이고, dist, finalized, parent 배열이 필요합니다.

- A*
  시간복잡도: 최악 O((V + E) log V)
  공간복잡도: O(V)
  이유: priority_queue를 사용하며, gScore, closed, parent 배열이 필요합니다.
       휴리스틱이 효과적이면 실제 방문 노드 수는 Dijkstra보다 줄어들 수 있습니다.

9. 주요 자료구조
============================================================
struct Node
- row, col 좌표를 저장합니다.
- 알고리즘에서 현재 위치, 이웃 위치, 부모 위치를 표현합니다.
- operator==를 통해 시작점/도착점 도달 여부와 같은 칸 여부를 비교합니다.

struct Cell
- CellType type으로 빈 칸, 벽, 시작점, 도착점, 가중치 셀을 구분합니다.
- int cost로 해당 칸으로 들어갈 때의 이동 비용을 저장합니다.

struct PriorityState
- Dijkstra와 A*의 priority_queue에 들어가는 상태입니다.
- row, col은 현재 좌표입니다.
- priority는 queue 정렬 기준입니다.
- distance는 시작점에서 현재 칸까지의 실제 누적 비용입니다.

struct SearchStats
- 마지막 탐색 결과를 저장합니다.
- found, visitedCount, pathLength, pathCost, elapsedMs를 통해 결과를 정량적으로 비교합니다.

vector<vector<Cell>> grid
- 전체 미로를 2차원 배열로 저장하는 핵심 자료구조입니다.
- grid[r][c]는 r행 c열의 칸을 의미합니다.

vector<vector<bool>> visited / finalized / closed
- 알고리즘에서 특정 칸을 이미 방문했는지 또는 최단 비용이 확정되었는지 저장합니다.

vector<vector<Node>> parent
- 각 칸이 어떤 이전 칸에서 왔는지 저장합니다.
- 도착점을 찾은 뒤 parent를 거꾸로 따라가면 최종 경로를 복원할 수 있습니다.

queue<Node>
- BFS에서 사용합니다.
- 먼저 들어온 칸을 먼저 처리하는 FIFO 구조입니다.

stack<Node>
- DFS와 랜덤 미로 생성에서 사용합니다.
- 나중에 들어온 칸을 먼저 처리하는 LIFO 구조입니다.

priority_queue<PriorityState>
- Dijkstra와 A*에서 사용합니다.
- 비용 또는 f-score가 가장 작은 후보 칸을 먼저 선택하는 데 사용합니다.

10. 주요 함수
============================================================
setup()
- 프로그램 시작 시 미로 파일 목록, 기본 미로, 화면 배치를 초기화합니다.

update()
- 방문 순서와 최종 경로 애니메이션을 일정 속도로 진행합니다.

draw()
- 미로, 우측 패널, 범례, 도움말을 화면에 출력합니다.

loadMazeFromFile()
- txt 미로 파일을 읽어 grid에 저장합니다.
- S, G, #, ., 숫자, W 문자를 각각 알맞은 CellType과 cost로 변환합니다.

createEmptyMaze()
- 테두리만 벽으로 막힌 빈 미로를 생성합니다.
- 사용자가 마우스로 직접 실험용 미로를 만들 수 있도록 합니다.

runBFS()
- queue 기반 BFS를 수행합니다.
- 방문 순서, parent 배열, 최종 경로, 실행 통계를 계산합니다.

runDFS()
- stack 기반 DFS를 수행합니다.
- 최단 경로 보장 여부와 방문 순서 측면에서 BFS와 비교할 수 있습니다.

runDijkstra()
- priority_queue 기반 Dijkstra 알고리즘을 수행합니다.
- 가중치 칸이 있는 경우 최소 비용 경로를 계산합니다.

runAStar()
- Manhattan Distance 휴리스틱을 사용하는 A* 알고리즘을 수행합니다.
- Dijkstra와 비교하여 방문 노드 수가 줄어드는 경우를 확인할 수 있습니다.

reconstructPath()
- parent 배열을 따라가며 도착점에서 시작점까지의 경로를 복원합니다.

computePathCost()
- 최종 경로의 총 이동 비용을 계산합니다.
- 시작점은 이동 비용에 포함하지 않고, 다음 칸으로 들어가는 비용부터 더합니다.

generateRandomMaze()
- stack을 이용한 Recursive Backtracking 방식으로 랜덤 미로를 생성합니다.
- 일부 빈 칸을 가중치 칸으로 바꾸어 비용 기반 탐색 비교가 가능하게 합니다.

screenToCell()
- 마우스 픽셀 좌표를 미로의 row/col 좌표로 변환합니다.

applyEditToCell()
- 현재 EditMode에 따라 클릭한 칸을 벽, 시작점, 도착점, 가중치 셀로 수정합니다.

drawMaze(), drawSidePanel(), drawLegend(), drawHelpBox()
- 미로, 통계 패널, 범례, 도움말을 각각 분리해서 출력합니다.

11. 창의적 구현 및 평가 기준 대응
============================================================
창의적 구현 항목 대응:
- BFS, DFS, Dijkstra, A* 네 가지 알고리즘을 하나의 프로그램에서 비교합니다.
- 단순 결과 출력이 아니라 방문 순서 애니메이션을 제공합니다.
- 가중치 셀을 지원하여 최단 칸 수와 최소 비용의 차이를 보여줍니다.
- 사용자가 마우스로 벽, 시작점, 도착점, 가중치 셀을 직접 편집할 수 있습니다.
- 랜덤 미로 생성 기능을 제공합니다.
- 방문 노드 수, 경로 길이, 총 비용, 실행 시간 통계를 제공합니다.

자료구조 및 알고리즘 항목 대응:
- BFS는 queue를 사용합니다.
- DFS는 stack을 사용합니다.
- Dijkstra와 A*는 priority_queue를 사용합니다.
- grid는 vector<vector<Cell>>로 구성했습니다.
- parent는 vector<vector<Node>>로 구성하여 최종 경로를 복원합니다.
- visibleVisited와 visiblePath를 별도로 두어 계산 결과와 화면 표시 상태를 분리했습니다.

프로젝트 완성도 항목 대응:
- 파일 로드, 알고리즘 실행, 애니메이션, 통계 출력, 마우스 편집, 랜덤 생성 기능을 모두 포함합니다.
- 코드 주석을 변수, 함수, 반복문 목적이 드러나도록 보강했습니다.
- 잘못된 파일 형식이나 파일 누락 상황에서도 빈 미로를 생성하도록 방어 코드를 넣었습니다.
- 화면 우측 패널에서 사용법과 결과 통계를 함께 확인할 수 있습니다.

시간 및 공간복잡도 항목 대응:
- README에 각 알고리즘의 시간복잡도와 공간복잡도를 정리했습니다.
- 소스코드에도 BFS/DFS/Dijkstra/A* 함수 위에 복잡도와 동작 원리를 주석으로 작성했습니다.
- 우측 패널에도 핵심 복잡도를 표시하여 시연 중 설명할 수 있게 했습니다.

============================================================
12. 보고서 작성 시 포함하면 좋은 내용
============================================================
보고서에는 다음 항목을 포함하는 것이 좋습니다.

1) 프로젝트 개요
- OpenFrameworks 기반 미로 탐색 알고리즘 비교 시각화 프로그램임을 설명합니다.
- BFS, DFS, Dijkstra, A*를 비교한다는 점을 명확히 씁니다.

2) 자료구조 설명
- grid, Cell, Node, parent, queue, stack, priority_queue의 역할을 설명합니다.
- 각 자료구조가 어떤 알고리즘에서 왜 필요한지 연결해서 설명합니다.

3) 알고리즘 설명
- BFS는 queue 기반 너비 우선 탐색입니다.
- DFS는 stack 기반 깊이 우선 탐색입니다.
- Dijkstra는 누적 비용이 작은 노드를 먼저 선택합니다.
- A*는 누적 비용과 휴리스틱을 함께 사용합니다.

4) 플로우차트
- 프로그램 전체 흐름: setup -> 미로 로드 -> 사용자 입력 -> 알고리즘 실행 -> parent 복원 -> 애니메이션 표시
- BFS/DFS 흐름: 자료구조 초기화 -> start 삽입 -> 반복 탐색 -> goal 확인 -> parent 복원
- Dijkstra/A* 흐름: dist/gScore 초기화 -> priority_queue 삽입 -> 최소 후보 확장 -> 비용 갱신 -> parent 복원

5) 시간 및 공간복잡도
- V=rows*cols, E=인접 간선 수로 정의합니다.
- BFS/DFS는 O(V+E), Dijkstra/A*는 O((V+E)logV), 공간은 O(V)라고 설명합니다.

6) 실험 결과
- 같은 미로에서 네 알고리즘의 방문 노드 수, 경로 길이, 총 비용, 실행 시간을 비교합니다.
- 가중치 셀이 있을 때 BFS와 Dijkstra/A*의 결과가 달라지는 사례를 포함하면 좋습니다.

7) 느낀 점 및 개선점
- 단순한 미로 탐색도 자료구조 선택에 따라 탐색 순서와 결과가 달라진다는 점을 서술합니다.
- 향후 대각선 이동, 다양한 휴리스틱, 더 큰 미로, 경로 비교 그래프 등을 개선점으로 제시할 수 있습니다.

13. 발표 및 시연 영상 구성 제안
============================================================
5분 내외 시연 영상은 다음 순서가 좋습니다.

1) 프로그램 실행 화면 소개
- 미로 영역과 우측 패널을 보여줍니다.
- 우측 패널에 조작법과 통계가 표시된다는 점을 설명합니다.

2) BFS 실행
- 1 키를 눌러 BFS 방문 순서와 최종 경로를 보여줍니다.

3) DFS 실행

4) Dijkstra 실행

5) A* 실행
- 4 키를 눌러 A*의 결과를 보여줍니다.
- 목표 방향 휴리스틱을 사용하므로 방문 노드 수가 줄어들 수 있음을 설명합니다.

6) 마우스 편집 및 랜덤 미로

7) 결과 통계 비교
- 방문 노드 수, 경로 길이, 총 비용, 실행 시간을 비교합니다.
- 가중치 미로에서 BFS는 최단 칸 수를, Dijkstra/A*는 최소 비용을 중시한다는 점을 결론으로 제시합니다.

15. 문제 해결
============================================================
1) make RunRelease가 안 되는 경우
- 대소문자를 확인합니다. RunRelease의 R과 R은 대문자입니다.
- make Runrelease는 잘못된 명령입니다.
- 먼저 make가 성공했는지 확인합니다.

2) Xcode 프로젝트가 없는 경우
- OpenFrameworks Project Generator에서 Import 후 Update를 누르면 생성됩니다.
- OpenFrameworks 프로젝트는 환경마다 프로젝트 파일을 재생성하는 것이 안전합니다.

3) 미로 파일이 로드되지 않는 경우
- maze_easy.txt, maze_weighted.txt, maze_large.txt가 bin/data 폴더 안에 있는지 확인합니다.
- 파일 이름이 README와 코드의 파일명과 같은지 확인합니다.
- 파일이 없어도 프로그램은 빈 미로를 생성하도록 방어 코드가 들어 있습니다.

4) 화면에 미로가 너무 작거나 크게 보이는 경우
- 창 크기를 조절하면 updateLayout()에서 셀 크기를 다시 계산합니다.
- 미로 크기가 큰 파일을 사용하면 셀 크기가 자동으로 작아질 수 있습니다.

5) BFS와 DFS 경로가 같게 나오는 경우
- 미로 구조가 단순하거나 유일 경로 미로이면 두 알고리즘의 최종 경로가 같을 수 있습니다.
- 중요한 차이는 최종 경로뿐 아니라 방문 순서입니다.
- 벽과 가중치를 직접 추가해 여러 case를 실험하면 차이가 더 분명해집니다.

16. 최종 요약
============================================================
Maze Pathfinder Lab는 미로를 그래프로 해석하고, BFS, DFS, Dijkstra, A*의 동작을
시각적으로 비교하는 프로젝트입니다. queue, stack, priority_queue, 2D vector,
parent 배열을 명확히 사용하며, 방문 순서 애니메이션과 결과 통계를 통해 알고리즘의
차이를 확인할 수 있습니다. 또한 마우스 편집과 랜덤 미로 생성 기능을 통해 사용자가
직접 실험 조건을 만들 수 있으므로, 구현 완성도와 창의적 확장성을 함께 보여줄 수 있습니다.
